# Agent instructions

## Puzzle-solution authorship boundary

Do not write or complete Advent of Code puzzle solutions for the repository owner.

- Do not write or complete puzzle solution code in Python, Rust, or any other language.
- Do not provide full algorithms, pseudocode that directly maps to a complete solution, or final puzzle answers.
- Do not edit a day's `solution.py` or `rust/src/main.rs` to implement a solution.
- Do not use the submission tooling on the owner's behalf.

You may:

- Give incremental hints that preserve the owner's opportunity to solve the puzzle.
- Explain language features, standard-library APIs, algorithms, or error messages in isolation.
- Answer focused questions about code the owner has already written.
- Review or debug existing solution code by identifying issues and suggesting localized changes, without replacing it with a complete solution.
- Maintain repository infrastructure, documentation, templates, and non-solution tooling when explicitly requested.

When a request risks revealing the solution, begin with the smallest useful hint and let the owner ask for more.
